## Introduction
The `database-helpers` provide a convenient way to persist and remove test data from your database during integration and E2E tests. These methods interact directly with your database through the configured adapter in Better Auth.

## Saving a User
To persist a user to the database, use the `saveUser()` method. This method accepts a user object (usually generated by `test.createUser()`) and returns the saved user object with any database-generated fields.

```typescript
const ctx = await auth.$context;
const test = ctx.test;

const user = test.createUser();
const savedUser = await test.saveUser(user);
```

## Deleting a User
Use `deleteUser()` to remove a user by their unique ID. This is critical for maintaining test isolation and preventing database clutter.

```typescript
await test.deleteUser(savedUser.id);
```

## Managing Organizations
If the `organization` plugin is enabled, you can manage organizations through the test utilities.

```typescript
const org = test.createOrganization({
    name: "Engineering",
    slug: "eng"
});

const savedOrg = await test.saveOrganization(org);
await test.deleteOrganization(savedOrg.id);
```

## Organization Membership
You can programmatically add members to organizations using the `addMember()` helper.

```typescript
await test.addMember({
    userId: user.id,
    organizationId: org.id,
    role: "admin" // or "member", "owner", etc.
});
```

## Test Isolation Principles
Each test should ideally start and end with a clean state.
- **Independence:** Tests should not rely on data created by other tests.
- **Cleanup:** Always remove data after the test finishes, even if it fails.

## Cleanup Patterns
Using `try/finally` blocks ensures cleanup occurs regardless of the test outcome.

```typescript
let user;
try {
    user = await test.saveUser(test.createUser());
    // ... run tests ...
} finally {
    if (user) await test.deleteUser(user.id);
}
```

## Cascading Deletes
When deleting a user or organization, Better Auth usually handles the cleanup of related sessions and accounts automatically if your database is configured with cascading foreign keys. If not, you should manually delete dependent records using standard database tools or specialized cleanup functions.
