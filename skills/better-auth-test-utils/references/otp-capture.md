## Introduction
The `otp-capture` feature in the test utilities allows you to programmatically access One-Time Passwords (OTPs) generated by Better Auth for authentication, email verification, and password reset flows. This eliminates the need for complex email/SMS mocking.

## Enabling OTP Capture
OTP capture is disabled by default. You must enable it in the `testUtils` plugin configuration.

```typescript
import { betterAuth } from "better-auth";
import { testUtils } from "better-auth/plugins";

export const auth = betterAuth({
    plugins: [
        testUtils({
            captureOTP: true
        })
    ]
});
```

## How Passive Capture Works
When `captureOTP` is enabled, the test utilities **passively** intercept any OTPs being sent through Better Auth. This means:
1. It does **not** stop the actual delivery of the OTP (it will still call your email/SMS provider).
2. It makes the OTP available through the `getOTP()` method.

## getOTP Method
The `getOTP()` method retrieves the most recently generated OTP for a given identifier (email or phone number).

```typescript
const ctx = await auth.$context;
const test = ctx.test;

const identifier = "test@example.com";
// Trigger OTP generation (e.g., via auth.api.sendVerificationEmail)
const otp = await test.getOTP(identifier);
```

## clearOTPs Method
To prevent old OTPs from leaking between tests, use `clearOTPs()` to reset the internal capture state.

```typescript
beforeEach(async () => {
    const ctx = await auth.$context;
    await ctx.test.clearOTPs();
});
```

## Testing Verification Flows
You can test the entire user verification flow without needing access to a real mailbox.

```typescript
// 1. Sign up user
await auth.api.signUpEmail({ email, password, name });

// 2. Get captured OTP
const otp = await test.getOTP(email);

// 3. Verify user with OTP
await auth.api.verifyEmail({ email, otp });
```

## Testing Password Reset
Testing password reset becomes trivial with OTP capture.

```typescript
// 1. Trigger reset
await auth.api.forgetPassword({ email });

// 2. Capture OTP
const otp = await test.getOTP(email);

// 3. Complete reset
await auth.api.resetPassword({ email, otp, newPassword: "new-secure-password" });
```

## Testing OTP Sign-In
If your application uses OTP-based sign-in, you can automate it easily.

```typescript
// 1. Request sign-in OTP
await auth.api.signInOTP({ email });

// 2. Retrieve OTP
const otp = await test.getOTP(email);

// 3. Verify sign-in
await auth.api.verifyOTP({ email, otp });
```
